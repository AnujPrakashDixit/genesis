HTML : Creating Interactive Forms

1. The <form> and <label> Elements

Forms are used to collect input from the user. The <form> tag is
the main container for all form fields.

An <input> creates a field, but it's meaningless without a
<label>. The 'for' attribute of the label must match the 'id'
of the input to connect them. This is crucial for usability and
accessibility.

Example:
<form>
  <label for="firstName">First Name:</label>
  <input type="text" id="firstName">
</form>

2. The 'name' Attribute & Submitting

To submit a form, you need a submit button. The <button> tag
with type="submit" is the modern way to do this.

The name attribute is essential. It acts as the "data label" or "key"
for the information sent to the server. The 'id' is for the browser,
the 'name' is for the server.

Example:
<form>
  <label for="firstName">First Name:</label>
  <input type="text" id="firstName" name="firstName">

  <button type="submit">Submit</button>
</form>

3. Radio Buttons & Checkboxes

Radio Buttons (Select One):
Use <input type="radio"> when a user can only choose one option.
All radio buttons in a group must share the same 'name' attribute.

<input type="radio" id="sizeS" name="shirtSize" value="small">
<label for="sizeS">Small</label>

Checkboxes (Select Many):
Use <input type="checkbox"> for multiple selections.

<input type="checkbox" id="pep" name="toppings" value="pepperoni">
<label for="pep">Pepperoni</label>

4. Textarea & Select Dropdowns

Textarea (Multi-line Text):
For longer text input like comments or addresses. 'rows' and 'cols'
attributes control its size.

<label for="comments">Comments:</label>
<textarea id="comments" name="comments" rows="4"></textarea>

Select Dropdown (Long Lists):
For choosing one option from a long list. The <select> tag
contains multiple <option> tags.

<label for="country">Country:</label>
<select id="country" name="country">
  <option value="in">India</option>
  <option value="us">USA</option>
</select>

5. Specialized Input Types (HTML5)

HTML5 added smarter input types for better user experience.

- password: Masks the input with dots.
  <input type="password" name="userPass">

- number: Shows a number keypad on mobile and allows for 'min'
  and 'max' validation.
  <input type="number" name="age" min="18" max="99">

- date: Displays a calendar date picker in most browsers.
  <input type="date" name="dob">
