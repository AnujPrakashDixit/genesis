HTML : File Paths & Page Structure

1. The File Path System

A website is a collection of files (HTML, images, etc.) in folders.
A file path is the exact address the browser needs to find a specific
file.

Example Folder Structure:
my-website/
├── index.html
├── images/
│   └── logo.png
└── pages/
    └── contact.html

2. Relative vs. Absolute Paths

Relative Paths (Most Common): Directions from your current file.
- Same Folder: <a href="about.html">About</a>
- Go Down: <img src="images/logo.png">
- Go Up: <img src="../images/logo.png"> (from contact.html)

ALWAYS use relative paths for your own project files.

Absolute Paths (External Files): The full web URL.
- Use for linking to other websites or hosted files (e.g., Google Fonts).
<a href="https://www.google.com">Google</a>

3. The HTML Boilerplate

The standard starting template for every HTML file. It gives the
browser essential setup information.

<!DOCTYPE html>              - Use latest HTML standards.
<html lang="en">             - Sets language to English.
  <head>                     - Container for metadata (for browser).
    <meta charset="UTF-8">   - Ensures characters display correctly.
    <meta name="viewport">   - Makes page mobile-friendly.
    <title>Page Title</title> - Sets the text in the browser tab.
  </head>
  <body>                     - Container for all visible content.
  </body>
</html>

Browsers are forgiving and will try to render pages without it, but this
can lead to bugs, poor mobile display, and bad SEO. Always use it.

4. Structuring Content (<div> & <span>)

<div> (The Generic Box):
A block-level container to group elements together for styling.
It creates a "box" around other tags.

<span> (The Inline Highlighter):
An inline container to style a small piece of text *within* a line,
without creating a line break.

id and class (Labels for Boxes):
- id: A UNIQUE identifier for one specific element. (#main-header)
- class: A REUSABLE classifier for multiple elements. (.profile-card)

5. Semantic Tags

HTML tags that give meaning to the content inside them. They act like
<div>s but tell the browser what the content *is*.

- <header>: Box for introductory content (logo, navigation).
- <main>: Box for the unique, main content of the page.
- <footer>: Box for closing content (copyright, contact info).
- <nav>: For a block of navigation links.
- <section>: For a thematic grouping of content.
